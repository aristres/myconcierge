# Uchiwai - 内祝いギフト推薦（MVP）

## 目的
忙しい新米パパ・ママが、出産内祝いの「候補探し」と「比較」を最短で終わらせる。

## 対象ユーザー
- 出産後〜1年の親（時間がない、選定疲れ、失礼が怖い）

## 提供価値
- 入力30秒 → おすすめ8件（理由つき）→ 楽天で購入へ

## MVP機能
1. 属性入力フォーム（シーン/予算/相手/好み/NG）
2. 推薦結果（8件、理由、楽天リンク、再提案）
3. 計測（form_submit / outbound_click / reroll）
4. 免責・広告表示・プライバシー

## 非目標（MVPではやらない）
- ログイン、履歴、ブックマーク
- 自動学習レコメンド
- 商品自動収集の完全自動化（最初は半自動＋レビュー）
- レビュー転載

## データ方針
- 商品は楽天APIで取得（実在担保）
- LLMはタグ付け・要約・理由生成のみ（商品を創作しない）

## 推薦方針（暫定）
- 必須条件：予算範囲、シーン一致、NG除外
- 加点：相手一致、好み一致、配送/個包装/定番度
- 出力：一致タグを理由として表示

## 法務・規約（運用方針）
- サイト上部に「広告」表示（常時）
- 楽天API利用時のクレジット表示（Rakuten Web Serviceガイドラインに従う）
- 楽天アフィリエイト禁止行為（DM送付・コメント欄貼り等）はやらない

## 開発優先順位
1. **Phase 1**: 入力フォーム UI
2. **Phase 2**: 楽天API連携 + データ取得
3. **Phase 3**: LLM推薦ロジック実装
4. **Phase 4**: 結果表示 + アフィリエイトリンク
5. **Phase 5**: アナリティクス計測
6. **Phase 6**: 法務表示 + デプロイ

## 技術スタック（候補）
- **フロントエンド**: Next.js, React, TailwindCSS
- **バックエンド**: Next.js API Routes または Express
- **LLM**: Claude API (Anthropic) または OpenAI
- **外部API**: 楽天商品検索API, 楽天アフィリエイトAPI
- **計測**: Google Analytics または Vercel Analytics
- **ホスティング**: Vercel

## 注意事項
- 楽天APIのレート制限に注意（1秒/1リクエスト等）
- アフィリエイトリンク生成時の規約遵守
- プライバシーポリシーの明記（GA使用時等）
